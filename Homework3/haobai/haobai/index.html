Design Rationale

This dashboard is meant to work like a small exploration tool instead of a set of separate charts. I wanted the user to start with the big picture first: which Pokémon are fast, which ones hit hard, and how those patterns change across types. The Attack vs. Speed scatterplot is the main context view because it keeps every Pokémon visible at once. The user can then click one Pokémon to inspect it more closely, or brush an area to focus on a group such as fast attackers, slow tanks, or Pokémon with more balanced stats.

The focus views are connected to the overview. The stat bar chart explains the selected Pokémon in a direct way, while the parallel coordinates chart gives a deeper comparison across HP, Attack, Defense, Special Attack, Special Defense, and Speed. The type distribution chart also works as a color reference because the same type colors are used across the dashboard and the bars are labeled by type. I kept the transitions simple on purpose: bars resize when the selected Pokémon changes, and points or lines fade when the focus group changes. These animations help users notice what changed without distracting from the data.
